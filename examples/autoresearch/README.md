# Autoresearch

A [Karpathy-style autoresearch](https://github.com/karpathy/autoresearch)
hill-climb on top of RLMFlow. **The agent is the researcher**: it reads
`train.py`, mutates it as a string, runs it via `run_experiment(source)`,
and decides what to try next. Wins get saved by `write_file('train.py',
winning_src)`. To run trials in parallel, the agent uses RLMFlow's
normal `delegate` / `wait`.

```
read train.py
 → src = read_file("train.py")
 → new = src.replace("LR: float = 3e-4", "LR: float = 1.5e-4")
 → r   = run_experiment(new) → {val_bpb, returncode, ...}
 → if val_bpb improved: write_file("train.py", new)
 → repeat (sequentially, or fan out via `delegate(...)`)
 → done(<summary>)
```

There is **no trial dir** to manage. Every call to
`run_experiment(source)` writes the source to `history/<n>_train.py`
and runs it from there (with `data/`, `prepare.py`, etc. symlinked in).
`history/ledger.jsonl` records every run; `list_runs()` reads it back.

## Layout

```
examples/autoresearch/
├── README.md
├── autoresearch.py        # driver — ~230 lines, no UI
└── tinker/                # example target: Tinker LoRA SFT on TinyStories
    ├── README.md
    ├── program.md         # agent-facing brief
    ├── prepare.py         # one-time data prep
    ├── train.py           # the file the agent reads + mutates
    └── data/              # generated by prepare.py
```

## Quick start

```bash
pip install -e .

export OPENAI_API_KEY=...
export TINKER_API_KEY=...

python examples/autoresearch/tinker/prepare.py     # one-time data prep
python examples/autoresearch/tinker/train.py        # optional sanity check

python examples/autoresearch/autoresearch.py \
    --target examples/autoresearch/tinker \
    --budget-s 300 --rounds 20 --model gpt-5
```

Output goes to `runs/autoresearch/`:
- `train.py` — the agent's current best (saved with `write_file`).
- `history/<n>_train.py` — every source string that was ever run.
- `history/ledger.jsonl` — `{n, ts, val_bpb, returncode, ...}` per run.
- `session/` — the agent's per-step transcript (root + each child).
- `viewer.html` — static replay.

## Adding your own target

Write a `train.py` whose last stdout line is `val_bpb: <float>` and
that exits 0 on success, plus a `program.md` agent brief. Drop them
in a directory and pass it via `--target`. The driver is
target-agnostic.
