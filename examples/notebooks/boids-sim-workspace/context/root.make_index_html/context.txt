Build a runnable browser-based Boids simulation with plain HTML, CSS, and JavaScript.

Global constraints:
- No build tools. No external libraries or CDNs.
- No ES modules. Use classic script tags only: <script src="..."></script>.
- The main runnable interface must be index.html at project root.
- Files to create: index.html, style.css, boids.js (all at project root).
- Use a dark color background.
- Render hundreds of colorful boids on a 2D <canvas> that fills the viewport.
- Do not add configuration controls or extra UI — just the canvas.
- The app should run when index.html is opened in a browser.

Verification expectations (the parent agent will check later):
- index.html includes a link to style.css and a script tag for boids.js (non-module).
- The <canvas> appears before the script tag so the script can bind to it without waiting.
- boids.js should have no obvious syntax errors and should not reference undefined globals.
- All files must exist after your write operation.

You are allowed (and expected) to write the target file yourself with write_file(path, content) and then return a concise status message.

Target file: index.html
Requirements:
- Minimal HTML5 document with:
  - <meta charset="utf-8"> and <meta name="viewport" ...>
  - <title>Boids Simulation</title>
  - A single <canvas id="boidsCanvas" aria-label="Boids simulation"></canvas> in <body>.
  - Link to "style.css" in <head>.
  - A single classic (non-module) <script src="boids.js"></script> placed at the END of <body>, AFTER the canvas.
- No other scripts or external resources.
- Keep markup minimal and clean.
